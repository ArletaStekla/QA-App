Please remember that you are allowed to grant admin rights only for applications that are STANDARD.



Please remember that each time you are sending ticket to a resolving team, it is your duty to collect all necessary details such as:

  • contact details for the user, location;
  • user is connecting to (LAN cable, wi-fi, home network);
  • device name/server name if user is on Citrix or uses a Citrix application such as Oracle, SAP;
  • exact website/SharePoint site address (should be typed in the description, not pasted on the screenshot);
  • screenshots of an error message and exact issue description (pasting an email from user/chat transcription is not enough!);
  • number of users affected and affected logins examples (if not all users at site are affected, it would be perfect to provide examples of IDs of users affected and not affected by the problem for comparison);
  • troubleshooting performed – follow the steps from the knowledge article even if there are multiple users affected.


Please remember that each time you are sending ticket to a resolving team or save it on your own queue, it is mandatory to link a KA on an Assisted profile.


As a reminder:
  • We close ticket from our end within the first contact – FTF = KA on Core or Assisted profile (preferred Core one);
  • We save ticket (using Defer or Save button) on the pending = KA on Assisted profile,
  • We send ticket internally - escalate it further (using Internal button) = KA on Assisted profile.
